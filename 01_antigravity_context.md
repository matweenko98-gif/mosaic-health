# Общий контекст проекта для ИИ (Antigravity Rules)

## 1. Описание приложения
* **Название проекта**: Мозаика Здоровья (Веб-приложение / PWA)[cite: 7].
* **Сфера**: Восстановительная медицина, кинезиотерапия, гиревое дыхание, трекинг здоровья[cite: 7].
* **Цель текущего этапа**: Создание полностью интерактивного прототипа из **двух экранов** для демонстрации заказчику[cite: 7]. Фокус на логике взаимодействия, архитектуре состояний и юзабилити[cite: 7]. Дизайн на данном этапе — схематичный (wireframe/черно-белый/светло-серый), используются заглушки для медиафайлов[cite: 7].

## 2. Технологический стек прототипа
* **Фронтенд**: React (с использованием Vite или встроенной сборки Antigravity)[cite: 7].
* **Стилизация**: Tailwind CSS (или базовые адаптивные стили) в упрощенном макете без финального UI-дизайна[cite: 7].
* **Бэкенд**: На данном этапе **ОТСУТСТВУЕТ**[cite: 7]. Все данные хранятся локально в React State (локальном состоянии) для демонстрации интерактивности "на лету"[cite: 7].

## 3. Золотые правила для Вайб-Кодинга (Правила для ИИ)
1. **Фокус на логике, а не на дизайне**: Не трать ресурсы на подбор сложных градиентов, анимаций и кастомных шрифтов[cite: 7]. Используй стандартную сетку, аккуратные отступы (padding, margin) и простые серые блоки-заглушки вместо картинок/видео[cite: 7].
2. **Интерактивность в приоритете**: Все кнопки, переключатели, формы ввода и модальные окна должны РАБОТАТЬ[cite: 7]. Клиент должен иметь возможность покликать прототип и увидеть изменения данных[cite: 7].
3. **Единый источник правды (State)**: Свяжи экраны между собой через общее состояние React (в родительском компоненте или через Context)[cite: 7]. Действие на Главном экране (например, завершение тренировки) должно мгновенно отражаться на экране Профиля (в истории тренировок)[cite: 7].
4. **Адаптивность под Mobile-First**: Верстка должна идеально смотреться на смартфонах, так как финальное приложение планируется как PWA для установки на телефон[cite: 7].
5. **Понятный код**: Пиши чистый, модульный код[cite: 7]. Разделяй интерфейс на логические компоненты[cite: 7].
